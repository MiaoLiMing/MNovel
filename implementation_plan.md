### 目标架构

```text
Android / iOS App
        │ 临时 HTTP :80（公网 IP）/ 后续 HTTPS :443（域名）
        ▼
宝塔 Nginx（证书、限流、反向代理）
        │ 127.0.0.1:18000
        ▼
Docker Compose / FastAPI（单 worker）
        │
        ▼
/opt/mnovel/data（宿主持久化目录）
├── mnovel.db
└── content-cache/
```

不再部署 Caddy，避免与宝塔 Nginx 争用 80/443。FastAPI 不开放公网端口，只由本机 Nginx 访问。公网 IP 阶段允许自用 App 经 Nginx 的 80 端口临时访问；该链路为明文传输，不承载高价值密钥或敏感内容，并在域名到位后立即切换 HTTPS。

### 验收标准

- 公网只能访问 80/443，不能直接访问 FastAPI 容器端口。
- 部署联调阶段通过服务器本机或 SSH 转发访问 `/api/v1/health` 并返回 200。
- 正式上线时 `https://<域名>/api/v1/health` 返回 200，TLS 证书有效。
- 未携带有效令牌的受保护请求返回 401/403。
- 容器重启、服务器重启后服务自动恢复，SQLite 数据不丢失。
- IP 联调包只对指定服务器 IP 放行 HTTP；域名上线包只连接 HTTPS，并移除明文例外。
- 备份可以在空目录中实际恢复并通过核心接口验证。

## 1. 项目目标

在 `D:/work/project/AP/MNovel` 中从零构建一套自用、无广告、支持 Android 与 iOS 的小说、短剧和视频聚合 App。小说与媒体均为一级核心业务，覆盖客户端、服务端、数据存储、内容源适配、阅读器、原生播放器、离线缓存、备份同步与自动化测试。

核心体验闭环：

```text
导入书源 → 聚合搜索 → 去重合并 → 查看详情 → 加入书架
→ 获取目录 → 阅读与缓存 → 保存进度 → 手动/自动换源

导入合法媒体源 → 聚合短剧/视频 → 查看详情与剧集
→ 收藏/续看 → 原生播放 → 预加载下一集 → 保存播放进度
```

产品底部固定四个主菜单：

```text
书架｜书城｜分类｜我的
```

`书架`和`书城`顶部固定频道 Tab：

```text
小说｜短剧｜视频
```

底部导航负责产品功能域，顶部频道负责内容类型；切换顶部频道不改变当前底部页面。

阅读器底部固定四个操作：

```text
目录｜夜间｜设置｜换源
```

## 2. 边界与原则

- 以私人阅读和观看、低资源占用和本地优先为目标，不加入广告、签到、金币、会员或评论社区。
- 不内置、不托管、不分发未经授权的小说、短剧、影视内容或具体侵权内容源；提供通用内容源协议、导入能力和本地演示数据。
- 用户导入第三方内容源时，由用户自行确认其版权、播放授权、DRM 许可、服务条款与访问权限。
- 默认单用户部署，不引入不必要的微服务、消息队列、Elasticsearch 或 Redis。
- 客户端优先读取本地缓存，服务端聚合请求采用并发、超时、限流、熔断和短期缓存。
- 所有敏感配置只进入环境变量或系统安全存储，不提交密钥。

## 3. 技术选型

### 3.1 移动端

- Flutter、Dart
- Riverpod：业务状态、依赖注入与可测试状态管理
- GoRouter：底部 ShellRoute、详情页、阅读器和播放器导航
- Dio：网络请求、取消、重试、拦截器与分批聚合结果
- Drift / SQLite：书架、收藏、阅读/播放进度、章节缓存索引、下载任务与设置
- Freezed、json_serializable：不可变模型、联合状态与 JSON 契约
- Android Media3 ExoPlayer：HLS/DASH、自适应码率、字幕、媒体会话、下载与 Widevine
- iOS AVPlayer / AVKit：HLS、系统播放控制、画中画、AirPlay 与 FairPlay 接入点
- 自有 `media_player` Flutter Platform Interface：统一 Android/iOS 播放器能力，避免业务层绑定第三方播放器插件
- flutter_test、integration_test、golden tests

### 3.2 服务端

- Python 3.12、FastAPI、Pydantic v2
- SQLAlchemy 2 Async、Alembic
- SQLite 默认运行；通过统一数据库层支持 PostgreSQL
- HTTPX：异步书源请求
- selectolax / lxml：HTML 解析与正文清洗
- RapidFuzz：书籍去重、标题与章节匹配
- 媒体元数据规范化、剧集匹配、播放线路健康检查和授权令牌透传
- pytest、pytest-asyncio、respx

### 3.3 工程化

- Flutter workspace / Melos 管理移动端、设计系统、播放器接口与共享 Dart 包
- Ruff、mypy 管理 Python 质量
- dart format、flutter analyze 管理 Dart 质量
- Dockerfile 与 docker-compose 提供本地服务端部署
- GitHub Actions 运行静态检查、单元测试与构建检查

## 6. 数据模型

主要实体：

- `Book`：规范化书籍，含书名、作者、封面、简介、分类、状态、字数。
- `BookEdition`：某本书在某个书源中的映射、URL、最新章节和质量指标。
- `Chapter`：统一章节索引与标题，不长期保存未授权远端正文。
- `Source`：书源配置、启用状态、优先级、能力、健康状态和最近检测结果。
- `SourceRule`：搜索、详情、目录、正文、分类的请求及解析规则。
- `ShelfItem`：书架记录、当前源、分组、置顶和更新状态。
- `ReadingProgress`：章节、章节内位置、阅读模式和更新时间。
- `Bookmark`：章节、段落、选中文字、备注。
- `ChapterCache`：本地缓存元数据、内容哈希、来源和过期时间。
- `DownloadTask`：下载范围、进度、状态、错误与重试信息。
- `CategoryMapping`：不同书源分类到应用统一分类的映射。
- `ReaderSettings`：主题、字号、行高、边距、翻页、屏幕和辅助设置。
- `BackupRecord`：备份版本、目标、校验和与恢复时间。
- `ContentChannel`：`novel`、`shortDrama`、`video` 三种顶层内容频道。
- `MediaTitle`：规范化短剧或视频，含名称、海报、简介、类型、年份和状态。
- `MediaEdition`：同一媒体在不同内容源中的映射、剧集数量、更新时间和质量指标。
- `Season` / `Episode`：季、集、时长、封面、简介、排序和发布状态。
- `PlaybackSource`：播放线路、协议、清晰度、字幕、请求头、DRM 与有效期元数据。
- `WatchProgress`：媒体、剧集、播放位置、总时长、最后观看时间和设备。
- `MediaDownload`：离线媒体、清晰度、分片、许可证状态、占用空间和过期时间。
- `FavoriteItem`：统一收藏模型，可引用小说、短剧或视频。

## 7. 功能范围

### 7.1 书架

- 顶部固定 `小说 / 短剧 / 视频` 频道 Tab，并分别记忆上次选择和滚动位置。
- 小说频道支持网格 / 列表、分组、排序、筛选与搜索。
- 短剧和视频频道展示收藏、续看、更新集数、下载状态和播放进度。
- 小说展示阅读进度、未读章节、更新状态、当前书源。
- 点击继续阅读，长按进入批量管理。
- 支持批量更新、删除、下载、分组和小说换源/媒体换线路。
- 启动时分频道增量检查；允许手动检查全部更新。

### 7.2 书城

- 顶部固定 `小说 / 短剧 / 视频` 频道 Tab；频道内榜单、筛选和卡片结构独立。
- 搜索框默认搜索当前频道，并允许在结果页切换全域搜索。
- 小说按书名与作者聚合去重；媒体按名称、年份、季数和别名聚合去重。
- 小说提供热门、完结、新书；短剧提供热播、完结、上新；视频提供电影、剧集、综艺等发现模块。
- 聚合结果展示可用内容源数量、更新时间、章节/集数和响应质量。
- 详情页支持加入书架/收藏、继续阅读/播放、下载、换源/换线路。
- 无真实远端榜单时使用本地演示数据，不伪造线上数据。

### 7.3 分类

- 分类首页按小说、短剧、视频分区，不重复底部导航。
- 小说支持男生、女生、出版及题材、背景、角色、风格、状态、字数、更新时间筛选。
- 短剧/视频支持类型、地区、年份、状态、语言、集数和更新时间筛选。
- 统一分类映射与多条件组合查询。
- 筛选条件可重置、持久化，并支持下拉刷新与分页。

### 7.4 我的

- 阅读与观看统计、收藏数量、缓存占用和同步状态。
- 内容源管理：区分书源和媒体源，支持导入、编辑、启停、排序、测试、批量检测和导出。
- 下载管理：任务状态、暂停、继续、取消、失败重试和空间提示。
- 阅读设置、缓存管理、导入导出、隐私设置与关于。
- WebDAV 备份与恢复：书架、进度、书签、设置、书源配置和搜索历史。

### 7.5 阅读器

- 默认上下滚动，支持上一章、下一章和章节连续加载。
- 沉浸态、操作态与单面板态；同一时间最多打开一个底部面板。
- 目录：当前章节自动定位、正倒序、标题搜索、分卷降级、已读与缓存状态。
- 夜间：日间、夜间、跟随系统；夜间不使用刺眼纯白正文。
- 设置：亮度、字号、字体、主题、行高、段距、页边距、首行缩进、屏幕常亮。
- 换源：展示匹配度、章节数、更新时间、速度、稳定性；支持手动与失败自动换源。
- 阅读进度保存章节索引及章节内偏移，不只保存总百分比。
- 当前章、前一章、后三章智能缓存；离线读取优先。
- 书签、系统 TTS、分享章节标题、刷新本章和正文问题标记。
- 错误状态明确区分网络失败、内容为空、源失效和章节不存在。

### 7.6 原生媒体播放器

- 短剧使用竖屏剧集播放器，支持上下滑动切集、播放/暂停、进度、倍速和选集。
- 维护最多三个活动播放器槽位：上一集、当前集、下一集；滑动后立即回收远端槽位。
- 下一集元数据和首段预加载，避免连续播放黑屏；根据网络和省流模式调整策略。
- 视频支持全屏、横竖屏、字幕、音轨、清晰度、画中画、后台音频和系统媒体控制。
- Android 使用 Media3 ExoPlayer；iOS 使用 AVPlayer/AVKit；Flutter 层仅依赖统一播放器接口。
- 播放失败明确区分网络、地址过期、解码、DRM、地区限制和内容源失效。
- 支持播放线路切换并恢复同一时间点；播放进度按节流策略本地保存并异步同步。
- 离线下载仅处理获得授权的媒体，支持暂停、恢复、空间检查、许可证状态和过期清理。

### 7.7 内容源与聚合引擎

- 定义统一能力：搜索、详情、目录、正文、分类与健康检查。
- 支持通用 JSON API 规则和 HTML/CSS 选择器规则。
- 请求规则支持 URL 模板、方法、查询、请求头、字符集、分页和超时。
- 解析规则支持字段选择、属性读取、正则替换、文本清洗和 URL 补全。
- 并发上限、每源限速、超时、一次重试、失败冷却与健康评分。
- 标题/作者规范化、模糊去重、章节序号解析和标题相似度匹配。
- 正文清洗去除导航、脚本、样式、多余空行和由用户规则明确指定的噪声。
- SSRF 防护：阻止环回、链路本地、私网和不安全协议；URL 导入二次校验。
- 定义 `MediaProvider`：搜索、详情、季集、播放线路、字幕、健康检查和刷新授权。
- 媒体聚合按标题、别名、年份、季和集进行规范化，避免错误合并同名作品。
- 播放地址和 DRM 令牌只短期缓存，不写入日志、备份或长期数据库。

- 关键操作有明确反馈，动画克制，并尊重系统“减少动态效果”设置。

## 10. 本地优先与同步策略

- 书架、收藏、阅读/播放进度、书签和设置先写入客户端本地数据库，再异步同步服务端。
- 冲突按实体处理：阅读进度取较新时间戳，书签按 ID 合并，设置按字段更新时间合并。
- 章节正文仅缓存在设备；默认按 LRU 和容量阈值清理。
- 媒体缓存和下载分开管理；临时分片按 LRU 清理，用户下载必须显式删除或按过期策略处理。
- WebDAV 备份生成版本化 JSON + 校验和，不上传服务端密钥或系统安全存储内容。
- 恢复前自动创建本地快照，恢复失败可回滚。

## 11. 性能目标

- 首屏冷启动在主流中端设备上目标不超过 2.5 秒。
- 缓存命中时打开正文目标不超过 150ms。
- 书架 500 本、目录 10000 章仍保持可滚动，使用虚拟列表。
- 聚合搜索按源流式归并或分批返回，首批结果目标 1.5 秒内可见。
- 移动端首包控制在合理范围，路由与大型面板按需加载。
- 图片按显示尺寸加载，启用懒加载和磁盘缓存。
- 服务端默认聚合并发可配置，单源有独立超时和限速。
- 短剧上下切集目标首帧等待不超过 500ms（下一集预加载命中时），滚动保持 60fps。
- 播放器池最多保留三个槽位，后台后按平台生命周期暂停并释放非当前播放器。
- 视频列表不创建播放器；只有进入可播放视口并达到阈值后才初始化当前播放器。

## 12. 安全与可靠性

- Pydantic / Freezed/json_serializable 模型校验输入和外部响应。
- 书源请求进行 URL、协议、DNS 解析结果和重定向链校验，防止 SSRF。
- WebDAV 凭据与服务端令牌存储于系统 Keychain / Keystore。
- 日志自动隐藏 Authorization、Cookie、密码和令牌。
- 导入文件限制大小、类型和嵌套深度；数据库写入使用事务。
- 数据库迁移可回退；备份恢复前校验版本与哈希。
- API 提供请求 ID、结构化日志和健康检查。
- 媒体播放接口只返回短时有效地址或授权结果；DRM 许可证、Cookie 和签名参数不进入日志或 WebDAV 备份。
- 明确处理 Android Widevine 与 iOS FairPlay 的授权边界，不尝试绕过 DRM 或平台内容保护。

- 左右滑到章节首尾时自动切换上一章/下一章，并避免误触连续切章。

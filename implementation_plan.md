## 一、现状审查结论

1. 当前 Flutter 主导航只有“收藏 / 发现 / 我的”三项，设计稿要求“书架 / 书城 / 分类 / 我的”四项。
2. 当前书城、书架、详情、阅读器和个人页的布局、字号、间距、配色及命名均与最新设计稿存在明显差异。
3. 分类筛选页、独立搜索页、完整章节目录页尚未按设计稿实现。
4. 阅读设置虽有基础能力，但面板结构、控件样式、主题命名和翻页选项需按新稿重构。
5. 阅读器顶部两个按钮仍为空回调；“所有能点击的都完善”尚未满足。
6. Flutter `ContentRepository` 目前以设备端直连为主，尚未完整接入现有 FastAPI 小说接口；`builtInContentSources` 为空时运行态会出现无来源空列表。
7. FastAPI 已具备目录、搜索、详情、章节、进度、收藏和书源的基础端点，但数据模型仍混有旧影视/短剧结构，且缺少新设计需要的首页分区、筛选、热搜、完整详情、来源排序和用户统计契约。
8. 当前工作区已有大量未提交修改，旧 `task.md`、`walkthrough.md`、`design-qa.md` 对应上一轮工作；本轮会更新这些交付文件，但不会回退既有实现。

## 二、视觉复刻基准

### 2.1 设计令牌

- 主色：珊瑚红 `#FF5A3C`。
- 辅助色：暖橙 `#FFB26B`。
- 暖白背景：`#FFF7F2`。
- 浅米色背景：`#F5EFE6`。
- 主文字：`#1F1F1F`。
- 次文字：`#666666`。
- 分割线：`#E9E4DC`。
- 不使用渐变；依靠纯色、细边框、圆角和克制阴影建立层次。
- 中文字体优先使用系统无衬线字体，并配置 `PingFang SC / Noto Sans CJK SC / Microsoft YaHei` 回退。
- 图标优先选用 Flutter 自带的 Material/Cupertino 图标；不使用 Emoji、文字字符或手绘 SVG 代替图标。

### 2.2 响应式基线

- 主要视觉验收视口：390 × 844。
- 补充验收：360 × 800、430 × 932、768 × 1024。
- 移动端保持设计稿信息密度；平板使用最大内容宽度和自适应列数，不做简单横向拉伸。
- 正确处理状态栏、安全区、键盘弹起、底部手势区和横屏阅读。

### 2.3 图片资产

- 逐一准备书城横幅、小说封面、列表封面和头像等独立位图资产，保持统一的东方奇幻/网文视觉方向。
- 优先复用项目内符合设计稿的现有封面；缺失资产使用 Image Gen 单独生成，并在写入项目后检查裁切、清晰度和色调。
- 不从整张设计板中直接切割并拉伸小图，不保留占位图。

## 四、实施方案

### 阶段 1：冻结契约与建立设计系统

1. 在 `task.md` 建立本轮任务清单，记录旧改动保护边界。
2. 建立统一的颜色、字号、圆角、间距、阴影、动画时长和触摸区域令牌。
3. 建立可复用的页头、底部导航、小说封面、标签、状态徽标、来源行、书籍列表行、分组标题、空态/错误态和骨架屏组件。
4. 增加明确的页面路由和结果回传，避免页面间以空回调或 SnackBar 假交互代替真实功能。

### 阶段 2：重构 Python 后端小说域

1. 将小说聚合域从旧的通用影视结构中分离，新增严格类型：
   - 小说摘要、小说详情、作者、标签、连载状态、评分、字数、章节数。
   - 书源、来源健康状态、优先级、章节映射和来源切换结果。
   - 搜索历史/热搜、筛选条件、首页分区、书架项、阅读历史、阅读进度和用户统计。
2. 保留现有 `/api/v1/mnovel` 前缀，扩充并兼容以下能力：
   - 首页分区、频道、分类元数据和筛选结果。
   - 搜索建议、热搜、搜索结果。
   - 小说详情、来源列表、章节目录、章节正文。
   - 收藏/书架、阅读历史、进度同步和统计。
   - 书源列表、健康检查、启停、排序、添加、编辑和删除。
3. 统一错误结构、分页、超时、缓存和幂等写入；不再用静态演示成功结果冒充真实来源。
4. 保留 JSON/JS 自定义来源能力，并把来源解析封装为适配器；非法 URL、内网地址、超时和单源失败均隔离处理。
5. 对现有数据库做兼容迁移，保留已有收藏、进度和来源数据。

### 阶段 3：接通 Flutter 数据层

1. 新增强类型 API 客户端、DTO 与领域模型映射，统一超时、重试、取消、错误提示和缓存策略。
2. 书城、搜索、分类、详情、目录、正文、书源和个人统计全部接入 FastAPI。
3. 保留设备端本地缓存和离线阅读能力；收藏、进度和设置采用本地即时写入，再与后端同步，避免弱网阻塞 UI。
4. 对旧 SharedPreferences 键和已收藏数据做向前兼容迁移。
5. 运行态不使用硬编码设计稿数据；视觉测试使用可重复的测试仓库/固定夹具。

### 阶段 4：逐屏复刻 Flutter UI

1. 先完成全局主题、四栏导航和页面过渡。
2. 按“书架 → 书城 → 分类 → 搜索 → 详情 → 目录 → 阅读器/设置 → 书源管理 → 我的”顺序实施。
3. 每屏同时完成正常、加载、空、失败、重试和选中状态。
4. 动画保持短促自然：底栏切换、筛选选中、展开收起、底部设置面板、目录定位和轮播均使用统一曲线。
5. 清查所有 `onTap`、`onPressed`、菜单项、图标按钮和列表项；禁止遗留空闭包或只显示“已准备”的假完成提示。

### 阶段 5：资产与细节校准

1. 将生成/复用的独立图片资产接入真实卡片并添加加载失败回退。
2. 校准设计稿中的视觉参数：页面边距、列表行高、封面比例、图标尺寸、字重、分割线、圆角、阴影和底部按钮高度。
3. 校准滚动吸附、键盘遮挡、安全区、长标题、省略、超长简介和大章节列表性能。
